# Submission checklist

## Required
- [ ] Team name
- [ ] Names of all team members
- [ ] Public GitHub repo link
- [ ] Weave project / trace link
- [ ] 2–3 sentence summary
- [ ] Clear description of what the project does
- [ ] Clear explanation of how it is built
- [ ] List of sponsor tools used
- [ ] List of protocols/frameworks used (MCP, A2A, etc. if applicable)
- [ ] X and/or LinkedIn handles

## Strongly recommended
- [ ] <2 min screen recording demo
- [ ] 1 slide max, maybe 2
- [ ] A few screenshots/gifs in case live demo is flaky
- [ ] Backup script for offline/demo-safe mode

## 3-minute live demo outline
- `0:00–0:20` — problem + who it's for
- `0:20–2:20` — live demo
- `2:20–2:45` — how it works / architecture
- `2:45–3:00` — why it matters + sponsor usage

## Judge-facing checklist
- [ ] The problem is obvious in the first 20 seconds
- [ ] The demo visibly works
- [ ] The agent behavior is easy to understand
- [ ] W&B Weave usage is visible, not hidden
- [ ] Sponsor/tool usage is explicit
- [ ] There is one memorable line about why this is new/useful

## Nice final-day assets
- [ ] README updated with real project description
- [ ] Architecture diagram or simple flow chart
- [ ] Demo script in bullets
- [ ] Social post draft for after the event
